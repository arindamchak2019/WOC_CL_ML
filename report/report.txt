Project Report
Event: Winter of Code (WOC) 
Project: Build Your Own Machine Learning Library and a Competition
1)	Kaggle Competition
To approach the specific problem of classification, I tried Logistical Regression first, without Regularization, which was prone to overfitting, which I tested by splitting the train data into cross validation set (20%) and train set (80%). I then experimented with different lambda values, for each case. I found that lambda value 3, and 9.5 gave pretty good accuracy while keeping cost functions of both sets low. Here lambda is Regularization factor.
I had 2 approaches for regularization in my 2 submissions to the competition. One took trial and error approach for lambda, testing different values of learning rate and lambda.
My second approach was just to keep a fixed lambda= 3. It seems that this one worked better, as a greater lambda is prone to underfitting. 
I had thought of using Neural Networks for this competition, but decided not to at the end moment, for my prior tries showed a lesser accuracy.
